# MOTION ARSENAL — полный арсенал беспоук-монтажа (гориз. 16:9 и верт. 9:16)

Собрано из боевых проектов. Набор ПРОВЕРЕННЫХ приёмов и кода, чтобы больше НЕ изобретать
заново и НЕ повторять ошибки. Читать вместе с `DIVERSITY_LAWS.md`.

> **Цвет акцента/обводки — из `BRAND_PROFILE.md`** (для V.CODE оранж `#EA5920`, НЕ лайм).
> Везде ниже, где написано «лайм», подставляй бренд-акцент клиента.

Работает и для говорящей головы (talking-head, 16:9), и для вертикальных стоков (9:16) —
принципы одни.

## 0. ГЛАВНЫЕ ЗАКОНЫ (жёстко, нарушение = брак)

1. **НИКАКОЙ СТАТИКИ.** Всё двигается всегда. Вставки — только ВИДЕО-стоки (не фотки).
   3D крутится всю сцену. У любой вставки есть движение: видео само, либо ken-burns/пан
   (crop с `t`), либо вращение 3D, либо анимация графики. Замерший кадр = брак.
2. **ФОРМА КАЖДОЙ ВСТАВКИ УНИКАЛЬНА.** Ни одна вставка не повторяет форму/раскладку
   соседей. Библиотека форм (см. §3) + раскладки (§2) + типы графики — 3 оси, по каждой
   нет повтора подряд, а формы вообще стараемся не повторять во всём ролике.
3. **ЛИЦО НЕ ПЕРЕКРЫВАЕМ.** Графика в центре → лицо РАЗМЫТО за ней. Сбоку → человека
   отодвигаем (reframe) в противоположную сторону. Никогда сырую графику на резкое лицо.
4. **КОНТРАСТ = ЗАТЕМНЕНИЕ ВИДЕО, а не тёмная плашка.** Под графикой затемняем/размываем
   само видео (по зоне или весь кадр). НИКАКИХ квадратных полупрозрачных «подложек» —
   они читаются как глюк. Обводка/свечение у самой графики — да, плашка — нет.
5. **МИКС форматов:** часть стоков на ФУЛЛ-ЭКРАН (с пан/зумом), часть — в формах. Не всё
   в формах и не всё фулл-скрин.

## 1. АРХИТЕКТУРА ПАЙПЛАЙНА (посегментная, надёжная)

`beats → drive → overlays(render) → compose(per-segment) → concat → audio → mux → (4K)`

- **beats.json** — манифест: `[{i,t0,t1, g:('comp'|'glb'|'stock',name,params), place|fmt, trans}]`.
  Тайминги из транскрипта; на каждый бит своя графика/сток по смыслу.
- **Посегментный рендер** (не один гигантский filter_complex): каждый бит → `seg_%03d.mp4`
  с ЕДИНЫМИ параметрами (1080×1920 верт. / 1920×1080 гориз., 30fps, yuv420p, libx264),
  потом concat demuxer. Надёжно, параллелится, нет рассинхрона.
- **Контиг-тайлинг для синхрона:** сегменты покрывают таймлайн БЕЗ дыр
  (`t0[0]=0`, `t1[i]=t0[i+1]`, последний = длине). Голос строим ПОСЕГМЕНТНО.

## 2. РАСКЛАДКИ (place)

`fs` фулл-экран графика · `center` крупно по центру над размытым+затемнённым им ·
`bottom` крупно снизу · `left`/`right` он отодвинут, графика сбоку в затемнённой зоне ·
`split` жёсткое деление · `lowbar` нижний титр (имя/CTA).
Затемнение зоны — мягкие фичер-градиенты (без жёстких краёв), центр — `gblur+eq=brightness=-0.34`.

## 3. БИБЛИОТЕКА ФОРМ ВСТАВОК (масками ffmpeg — для видео И фото)

15+ форм, у каждой маска (`shapes/<name>_mask.png`, серый feather) + акцент-обводка-кольцо
(`shapes/<name>_ring.png`, «внешняя − внутренняя фигура»):
**circle, oval, hexagon, pentagon, diamond, roundsquare, rrect, arch, tilt(наклон), tv,
parallelogram, strip(верт.полоса), band(гор.полоса), phone(мокап), smallcircle.**
Плюс `full` (фулл-экран) и `feather` (крупно с растворением краёв).

Композит формы: `media(cover w×h) → alphamerge(mask) → overlay(x,y) на затемнённый →
поверх overlay ring`. У видео пан не нужен (движется само); у крупного центра — лёгкий зум.

**Фулл-экран движение — ТОЛЬКО crop-pan (4 направления), НЕ zoompan zoom-out/pan.**
`scale=..., crop=W:H:x='(iw-W)*t/DUR':y=...` (zoompan zoom-out/пан-выражение даёт ЧЁРНЫЙ кадр).

Генератор форм (верт. адаптация): `pipeline/motion/make_shapes.py` (PIL: rounded_rect/
ellipse/polygon + GaussianBlur feather; кольцо = outer − inner, залито бренд-акцентом).
Композитор вставки: `pipeline/motion/shaped_insert.py` (клип → маска-форма → ring → на
затемнённый основной кадр).

## 4. ДВИЖОК ГРАФИКИ — беспоук-инфографика кодом, рендер в альфу

- Наш `motion/fx_engine.js` (canvas→PNG-альфа) уже даёт ~20 брендовых компонентов
  (stat, gauge, donut, bars, callout, dm/likes/toast, kinetic-slam, particles, ticker,
  citymap, steps, gridpop, profilecmp, moneycount, calendar, shine, lowerthird…).
  ПАЛИТРА для выбора — брать разное под смысл, мимо `USED_ANIM`.
- Для длинных/тяжёлых анимаций — рендер в **webm с альфой** (vp9 yuva420p):
  Playwright + headless_shell → скриншоты `omitBackground:true` →
  `ffmpeg -c:v libvpx-vp9 -pix_fmt yuva420p -auto-alt-ref 0`.
  **КРИТИЧНО:** при чтении такого webm — `-c:v libvpx-vp9` ПЕРЕД `-i`, иначе нативный
  vp9-декодер ВЫКИДЫВАЕТ альфу (графика станет чёрным прямоугольником).
- **Подложку-карточку НЕ рисуем** там, где можно затемнить видео (контраст — затемнением).
  Самая «заходящая» графика — анимированный линчарт «потолок → пробиваем». Таких побольше.

## 5. 3D (GLB) — реальные крутящиеся фигуры

- Источники GLB: Higgsfield `generate_3d` (tripo — платно, но по запросу/премиум),
  **MIND** MCP (`start_model_generation`/`animate_generated_model`), Sketchfab, готовые.
  Для БЕСПЛАТНОГО завода по умолчанию — процедурные three.js-фигуры (`three/three_fx`).
- **Децимация тяжёлых GLB:** `npx @gltf-transform/cli simplify --ratio .25 --error .002` +
  `resize --width 768` (56MB→~14MB). Грузится в обычный GLTFLoader.
- Рендер: three.js ESM + GLTFLoader в `glb_view.html` через `python3 -m http.server`
  (ES-модули требуют http), Playwright `--use-gl=swiftshader`, студийный HDR, ACESFilmic → webm-альфа.
- **Правила 3D:** центрировать ИЛИ аккуратно сбоку, влезать в кадр, крутиться ВСЮ сцену,
  уходить плавно. Каждую фигуру — максимум 1 раз за ролик.

## 6. WebGL / Spine / Lottie / шейдерные переходы

- WebGL-эффекты «как на видео»: React Bits / UIverse / 21st.dev / Three.js shaders — скачивать
  под тему, рендерить в webm-альфу тем же Playwright-пайплайном.
- Lottie: lottiefiles (бесплатно), `motion/lottie_render.js`. Spine/Rive — как оверлеи-акценты.
- Шейдерные переходы: `motion/transitions_gl.cjs` (125 gl-transitions). Он→фулл-экран сток —
  быстрый zoom-punch/whip/luma.

## 7. ЗВУК — музыка по секциям + ОСМЫСЛЕННЫЕ SFX

- Музыка: свой трек на ролик (или 4-5 по секциям для длинных), даккинг под голос
  `sidechaincompress(threshold≈0.03-0.06:ratio≈5-6)`.
- **SFX по смыслу бита:** `impact` на цифрах/сравнениях/3D, `ding` на CTA/позитиве,
  `whoosh` на стоках/переходах/появлениях. Не «вжух везде». Freesound.
- Голос: `dynaudnorm` + лимитер. Мастер `amix` + `alimiter/loudnorm I=-14:TP=-1.5`.

## 8. ГРЕЙД / ФИНАЛ

- Не грейдить дважды. Доп.грейд: `eq+colorbalance+vignette` (+лёгкое зерно — дорого CPU).
- 4K: `scale=3840:2160:flags=lanczos` (апскейл не даёт деталей, но даёт «4K»-лейбл).

## 9. ГРАБЛИ ОКРУЖЕНИЯ (проверено болью)

- **Фоновые ffmpeg реапаются ~2-3 мин** — длинные (грейд/энкод/4K) резать на куски ≤~90с
  ИЛИ foreground кусками; 4K по 50с + concat demuxer.
- **Быстрый seek базы:** перекодировать с частыми ключевыми (`-g 15 -keyint_min 15`).
- **Playwright копит память** — рендерить оверлеи батчами (≤12) со свежим браузером (OOM ~15-й).
- **30MB-лимит отдачи в чат** — хостить видео: Higgsfield `media_upload`→PUT байты curl'ом
  (`--cacert /root/.ccr/ca-bundle.crt`)→`media_confirm`→CDN-ссылка. Или пережать 2-pass < 30MB.
- **Коннекторы рвутся** — ретраить; get_cost НЕнадёжен — сверять по `balance`.
- **Gemini как «глаза»** (по возможности): прогонять секции, чинить до 8.5+.

## 10. АНАЛИЗ ГОТОВОГО РОЛИКА

- Higgsfield `video_analysis_create`→`video_analysis_status` (сцена-за-сценой),
  `virality_predictor` (хук/удержание/виральность).
- Свой скрин-скан чёрных кадров: кадр каждого сегмента, средняя яркость <10 = баг
  (так ловится чёрный сток от zoompan-zoom-out).
