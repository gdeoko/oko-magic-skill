# MOTION ARSENAL — полный арсенал беспоук-монтажа (гориз. 16:9 и верт. 9:16)

Собрано из боевого проекта «ролик Михаила Кадяева» (5:30, 4K). Это набор ПРОВЕРЕННЫХ
приёмов и кода, чтобы больше НЕ ИЗОБРЕТАТЬ заново и НЕ ПОВТОРЯТЬ одни и те же ошибки.
Читать вместе с `DIVERSITY_LAWS.md`. Код лежит в `pipeline/horizontal/`.

Работает и для говорящей головы (talking-head, 16:9), и для вертикальных стоков (9:16) —
принципы одни.

---

## 0. ГЛАВНЫЕ ЗАКОНЫ (жёстко, нарушение = брак)

1. **НИКАКОЙ СТАТИКИ.** Всё двигается всегда. Вставки — только ВИДЕО-стоки (не фотки).
   3D крутится всю сцену. У любой вставки есть движение: видео само, либо ken-burns/пан
   (crop с `t`), либо вращение 3D, либо анимация графики. Замерший кадр = брак.
2. **ФОРМА КАЖДОЙ ВСТАВКИ УНИКАЛЬНА.** Ни одна вставка не повторяет форму/раскладку
   соседей. Библиотека форм (см. §3) + раскладки (§2) + типы графики — 3 оси, по каждой
   нет повтора подряд, а формы вообще стараемся не повторять во всём ролике.
3. **ЛИЦО НЕ ПЕРЕКРЫВАЕМ.** Графика в центре → лицо РАЗМЫТО за ней. Сбоку → человека
   отодвигаем (reframe) в противоположную сторону. Никогда сырую графику на резкое лицо.
4. **КОНТРАСТ = ЗАТЕМНЕНИЕ ВИДЕО, а не тёмная плашка.** Под графикой затемняем/размываем
   само видео (по зоне или весь кадр). НИКАКИХ квадратных полупрозрачных «подложек» —
   они читаются как глюк. Обводка/свечение у самой графики — да, плашка — нет.
5. **МИКС форматов:** часть стоков на ФУЛЛ-ЭКРАН (с пан/зумом), часть — в формах. Не всё
   в формах и не всё фулл-скрин.

## 1. АРХИТЕКТУРА ПАЙПЛАЙНА (посегментная, надёжная)

`beats → drive → overlays(render) → compose(per-segment) → concat → audio → mux → (4K)`

- **beats3.json** — манифест: `[{i,t0,t1, g:('comp'|'glb'|'stock',name,params), place|fmt, trans}]`.
  Тайминги берём из транскрипта; на каждый бит своя графика/сток по смыслу.
- **Посегментный рендер** (не один гигантский filter_complex): каждый бит → `seg_%03d.mp4`
  с ЕДИНЫМИ параметрами (1920×1080/30fps/yuv420p/libx264), потом concat demuxer. Надёжно,
  параллелится, нет рассинхрона.
- **Контиг-тайлинг для синхрона губ:** сегменты должны покрывать таймлайн БЕЗ дыр
  (`t0[0]=0`, `t1[i]=t0[i+1]`, последний = длине). Иначе склейка «сожмётся» и звук уедет.
  Голос строим ПОСЕГМЕНТНО (вырезаем cc[t0:t1] и склеиваем) → всегда совпадает с видео.

## 2. РАСКЛАДКИ (place) — как человек и графика делят кадр

`fs` фулл-экран графика · `center` крупно по центру над размытым+затемнённым им ·
`bottom` крупно снизу (лицо сверху чистое) · `left`/`right` он отодвинут, графика сбоку в
затемнённой зоне · `split` жёсткое деление · `lowbar` нижний титр у самого низа (имя/CTA).

Затемнение зоны — фичер-градиенты `grad_left/right/bottom.png` (мягкие, без жёстких краёв),
центр — `gblur=sigma≈15,eq=brightness=-0.34`. Reframe человека: `scale=2208:1242,crop=1920:1080:X:100`
(X=288 сдвигает влево, X=0 — вправо).

## 3. БИБЛИОТЕКА ФОРМ ВСТАВОК (масками ffmpeg — работает для видео И фото)

15+ форм, у каждой маска (`shapes/<name>_mask.png`, серый feather) + лаймовая обводка-кольцо
(`shapes/<name>_ring.png`, полоса «внешняя − внутренняя фигура»):
**circle, oval, hexagon, pentagon, diamond, roundsquare, rrect, arch, tilt(наклон), tv,
parallelogram, strip(верт.полоса), band(гор.полоса), phone(мокап), smallcircle.**
Плюс `full` (фулл-экран) и `feather` (крупно с растворением краёв).

Композит формы: `media(cover w×h) → alphamerge(mask) → overlay(x,y) на затемнённый им →
поверх overlay ring`. Ken-burns/пан у видео не нужен (движется само); у крупного центра —
лёгкий зум.

**Фулл-экран движение — ТОЛЬКО crop-pan (4 направления), НЕ zoompan zoom-out/pan.**
`scale=2160:1215, crop=1920:1080:x='(iw-1920)*t/DUR':y=...` — 4 стороны. (zoompan с zoom-out
или пан-выражением даёт ЧЁРНЫЙ кадр — проверено, не использовать.)

Генератор форм: `pipeline/horizontal/make_shapes.py` (PIL: rounded_rect/ellipse/polygon +
GaussianBlur feather; кольцо = outer_mask − inner_mask, залитое лаймом).

## 4. ДВИЖОК ГРАФИКИ (anim.js) — беспоук-инфографика кодом, рендер в webm с альфой

- **anim.js** — детерминированный движок: `window.renderT(t)` гоняет TRACKS, easing,
  «живые» фоны, reframe, глитч, зерно. ~34 компонента: stat, gauge, donut, linechart,
  map_routes(маршруты), arrows, flowtree, timeline, compare, glassticker, glassquote,
  kinetic, title, name(нижний титр), device(телефон с экраном), pill, list*, subscribe,
  cta_title, photo3d, hex/split/card/billboard(картинка в форме).
- **Over-mode:** `P.over=true` → графика прозрачная в зоне (opos L/C/R, oscale, oy),
  `window.__NOBG` глушит фон. **Подложку-карточку НЕ рисуем** (глюк) — контраст даёт
  компоузер затемнением видео.
- **Рендер в webm с альфой:** Playwright + headless_shell → скриншоты каждого кадра с
  `omitBackground:true` → `ffmpeg -c:v libvpx-vp9 -pix_fmt yuva420p -auto-alt-ref 0`.
  **КРИТИЧНО:** при чтении такого webm в компоузере — `-c:v libvpx-vp9` ПЕРЕД `-i`, иначе
  нативный vp9-декодер ffmpeg ВЫКИДЫВАЕТ альфу (графика станет чёрным прямоугольником).
- Инфографика ПИШЕТСЯ ЗАНОВО под цифру/метафору (SVG getPointAtLength для линий/путей,
  анимация счётчиков, стрелки, диаграммы). Самая «заходящая» — анимированный линчарт
  «потолок → пробиваем». Таких — побольше.

## 5. 3D (GLB) — реальные крутящиеся фигуры

- Источники GLB: Higgsfield `generate_3d` (tripo — платно, надёжно; sam_3_3d дешевле но
  падает на тёмных рендерах), **MIND** MCP (`start_model_generation`/`animate_generated_model`
  — генерация и анимация 3D), Sketchfab, готовые.
- **Децимация тяжёлых GLB:** `npx @gltf-transform/cli simplify --ratio .25 --error .002` +
  `resize --width 768` → 56MB→~14MB, качество цело, грузится в обычный GLTFLoader.
- Рендер: three.js r160 ESM + GLTFLoader в `glb_view.html`, отдаётся через
  `python3 -m http.server` (ES-модули требуют http, не file://), Playwright `--use-gl=swiftshader`.
  Студийный HDR-энвайрон, лайм+холодный свет, ACESFilmic. → webm с альфой.
- **Правила 3D:** центрировать (offX=0) ИЛИ аккуратно сбоку, влезать в кадр (не обрезать
  справа), крутиться ВСЮ сцену (без заморозки), уходить плавно (без резкого исчезновения).
  Каждую фигуру — максимум 1 раз за ролик.

## 6. WebGL / Spine / Lottie / шейдерные переходы

- WebGL-эффекты и «как на видео»: React Bits / UIverse / 21st.dev / Three.js shaders —
  скачивать под тему, рендерить в webm с альфой тем же Playwright-пайплайном.
- Lottie: lottiefiles (бесплатно), рендер lottie-web в canvas → webm-альфа.
- Spine/Rive: rive-canvas / spine-player → webm-альфа. Использовать как оверлеи-акценты.
- Шейдерные переходы между кадрами: `pipeline/motion/transitions_gl.cjs` (gl-transitions).
  Для перехода «он → фулл-экран сток» — быстрый zoom-punch/whip/luma.

## 7. ЗВУК — музыка по секциям + ОСМЫСЛЕННЫЕ SFX

- Музыка: 4-5 треков по секциям (интро/корп/дарк/аплифт/CTA), кроссфейды, `volume≈0.28`,
  даккинг под голос `sidechaincompress(threshold=0.06:ratio=6)`.
- **SFX по смыслу бита** (не просто вжух везде): `impact` на цифрах/сравнениях/3D,
  `ding` на подписке/CTA/позитиве, `whoosh` на стоках/переходах/появлениях. Freesound.
- Голос: `dynaudnorm` + лимитер. Всё сводить `amix` + `alimiter=limit=0.96`.

## 8. ГРЕЙД / ФИНАЛ

- База уже с грейдом (breathing zoom+grade+vignette+grain) — НЕ грейдить дважды.
- Если нужен доп.грейд: `eq+colorbalance+vignette` (лёгкое зерно `noise` — дорого по CPU).
- 4K: `scale=3840:2160:flags=lanczos`. Апскейл 1080→4K не даёт деталей, но даёт «4K»-лейбл.

## 9. ГРАБЛИ ОКРУЖЕНИЯ (проверено болью)

- **Фоновые задачи РЕАПАЮТСЯ** через ~2-3 мин: длинные ffmpeg (грейд/энкод/4K) — резать на
  куски ≤~90с контента ИЛИ гонять в foreground кусками. 4K — по 50с, concat demuxer.
- **Быстрый seek базы:** перекодировать базу с частыми ключевыми кадрами
  (`-g 15 -keyint_min 15`) → seek мгновенный (иначе accurate_seek декодит от редкого
  keyframe = очень медленно).
- **Playwright chromium копит память** — рендерить оверлеи батчами (≤12) со свежим
  браузером, иначе OOM на ~15-й задаче.
- **Отдача 30MB-лимит в чат** — хостить видео: Higgsfield `media_upload`→PUT байты curl'ом
  (через `--cacert /root/.ccr/ca-bundle.crt`)→`media_confirm`→CDN-ссылка `d2ol7oe51mr4n9.cloudfront.net`.
- **Коннекторы (Higgsfield и др.) рвутся** — ретраить вызов; curl PUT байтов не зависит от
  коннектора. Higgsfield get_cost НЕнадёжен — сверять по balance, а не по preflight.
- **Gemini как «глаза»:** `gemwatch.sh <video> "<промпт>"` (curl, не urllib — иначе CF 1010;
  ротация ключей на 429; при 503 — модель перегружена, повторить). Прогонять секции,
  чинить до 8.5+.

## 10. АНАЛИЗ ГОТОВОГО РОЛИКА

- Higgsfield `video_analysis_create`(video_input_id) → poll `video_analysis_status`
  (сцена-за-сценой), и `virality_predictor` (хук/удержание/виральность).
- Свой скрин-скан на чёрные кадры: извлечь кадр каждого сегмента, среднюю яркость <10 =
  баг. Так пойман чёрный сток от zoompan-zoom-out.
