# Reels Machine v6 — контент-завод роликов под ключ

Из сценария → готовый вертикальный ролик 1080×1920, 15–50 сек, уровня студийного
монтажёра. Контент бесплатный (стоки, озвучка, 3D, инфографика, переходы — локально).
Платный ИИ используется ТОЛЬКО на обложку и карусели (Nano Banana Pro, безлимит-подписка).

> **v7 — ГЛАВНОЕ:** законы разнообразия + ИИ-обложка + **беспоук-арсенал монтажа**.
> Прочитать `reference/DIVERSITY_LAWS.md`, `reference/BRAND_PROFILE.md` и
> **`reference/MOTION_ARSENAL.md`** ДО сборки любого ролика. Нарушение законов = брак.
>
> **`reference/MOTION_ARSENAL.md`** — боевой арсенал: библиотека ФОРМ вставок (круг/
> телефон/ромб/шестиугольник/полоса/наклон/арка/TV/…) с бренд-обводкой, раскладки, НОЛЬ
> статики (всё видео и движется), лицо не перекрывать, контраст ЗАТЕМНЕНИЕМ видео (не
> плашкой), webm-альфа для тяжёлой графики, реальные 3D-GLB (децимация), SFX по смыслу,
> посегментный рендер, скрин-скан чёрных кадров. Формы: `motion/make_shapes.py` +
> `motion/shaped_insert.py`.

## ⚖️ ТРИ ЗАКОНА РАЗНООБРАЗИЯ (важнее всего — читать первыми)

Определение: **«анимация»** = ВСЁ, что накладывается на стоковый кадр — GLB/3D-фигуры,
3D-графика, инфографика кодом, кинетика, callout, штампы, партиклы, световые эффекты,
шейдерные переходы, оверлеи. Всё это подчиняется законам ниже.

### Закон 1. Каждый ролик собирается С НУЛЯ под свой сценарий
- Анимации, кадры, озвучка, музыка, SFX, грейд, обложка — **подбираются под анализ
  сценария и темы этого ролика** (шаг «Режиссёрский манифест»).
- **Ничего не берётся «с полки» готовым и не повторяется бездумно.** Инфографика и
  беспоук-графика **пишутся заново** каждый раз под конкретную цифру/метафору сценария —
  НЕ шаблон с подменённой цифрой/буквой. Ассеты (Lottie, gl-переходы, музыка, звук,
  кадры) **ищутся/скачиваются заново** под ключевые слова сценария.
- **Неизменны ТОЛЬКО бренд-константы** (см. `reference/BRAND_PROFILE.md`): голос озвучки,
  цвет субтитров, шрифт субтитров, лого, финальная карточка, палитра. Всё остальное —
  разное в каждом ролике.

### Закон 2. Каждый кадр уникален; кадры меняются каждые 3–5 сек
- **Ни один стоковый клип не повторяется внутри ролика.** На ролик 15–50 сек нужно
  **10–14 УНИКАЛЬНЫХ клипов** (не 3–5 по кругу). Кадр меняется каждые 3–5 сек.
- Кадры подбираются **по битам сценария** (каждая фраза → свой визуал) и стоят в
  **логичной нарративной последовательности** (проблема → процесс → результат → CTA).
- **Между роликами кадры тоже не повторяются** — дедуп по id через `USED_FOOTAGE`.
- **Бренд-нить (сугубо под нишу клиента):** бренд-кадры — НЕ глобальное правило. Они
  задаются индивидуально в `BRAND_PROFILE` каждого клиента под ЕГО нишу, и только если это
  уместно. У всех ниши разные — не переносить бренд-кадры/приёмы одного клиента на других.
  Нет заданных бренд-кадров — просто не вставлять.

### Закон 3. Память завода запрещает повторы
- Перед сборкой читать реестры: `reference/USED_FOOTAGE.md` (id кадров) и
  `reference/USED_ANIM.md` (приёмы, Lottie-запросы, gl-переходы, 3D-фигуры, типы
  инфографики, грейды, id музыки/SFX). **Исключить всё уже использованное.**
- Правило: приём/ассет не повторяется **3–5 роликов**; переходы и финалы — всегда разные;
  кадры — вообще не повторяются.
- После сборки **дописать** в оба реестра, что использовано. Это память, которая не даёт
  скатиться в шаблон.

## 🚀 ШАГ 0 — ОНБОРДИНГ БРЕНДА (один раз на клиента, до первого ролика)

Скилл УНИВЕРСАЛЬНЫЙ — работает для любого бизнеса/ниши/клиента. Перед запуском завода
надо один раз собрать паспорт бренда в `reference/BRAND_PROFILE.md`. Если он ещё не
заполнен под текущего клиента (или в нём стоят плейсхолдеры/пример V.CODE) — СНАЧАЛА
уточнить у пользователя и записать:
1. **Название бренда / бейдж** и ниша/сфера бизнеса.
2. **Логотип** — попросить файл `logo_hd.png` (PNG с прозрачностью). Из него же берём
   акцентные цвета (можно вытащить пипеткой). Логотип НЕ входит в скилл — его даёт клиент.
3. **Палитра** — акцентные цвета бренда (обычно из лого). Именно они идут в оверлеи,
   субтитры-выделение, обложку. (Ни в коем случае не «дефолтный зелёный».)
4. **Шрифт субтитров** (по умолчанию Союз Гротеск) и **голос** озвучки (муж/жен).
5. **CTA** и бренд-строку финалки.
6. **Бренд-кадры** — ключевые слова стока под нишу клиента (что «сшивает» ролик с брендом).
7. **Есть ли Higgsfield Ultra** (безлимит Nano Banana Pro) для ИИ-обложек — если нет,
   обложка собирается фолбэком (сток + текст), см. ниже.

Записать всё в `BRAND_PROFILE.md` (это ЕДИНСТВЕННЫЙ файл под клиента). Дальше завод
работает по данным этого клиента. Сменить клиента = переписать `BRAND_PROFILE.md` +
подложить его `logo_hd.png` и шрифт. Механика (движки, законы, пайплайн) — общая для всех.

## 🖼️ ОБЛОЖКА — полностью через нейросеть (Nano Banana Pro)

**Правило:** обложка ролика генерируется ЦЕЛИКОМ нейросетью, **включая текст внутри
обложки** — шикарная брендовая обложка одним изображением. Не собираем обложку из
стокового кадра + PIL-текста (это старый способ v5, оставлен только как аварийный фолбэк).

- Модель: **`nano_banana_pro`** (Google, на Higgsfield MCP) — «ultimate quality, text and
  diagrams», рендерит текст и держит 9:16. Требует **Ultra-подписку Higgsfield с безлимитом
  на генерацию картинок** (проверять `balance` → `subscription_plan_type: ultra`).
- Вызов: `generate_image(model='nano_banana_pro', params={aspect_ratio:'9:16', prompt:...})`,
  затем забрать URL результата (`job_display`/`show_generations`) и скачать curl-ом в `cover.jpg`.
- Промпт — БОГАТЫЙ и детальный (сцена, свет, объектив, глубина, фактура, эмоция), строго
  в БРЕНД-ЦВЕТАХ логотипа (см. BRAND_PROFILE: оранж `#EA5920` + белый + чёрный, БЕЗ
  зелёного), негативное пространство под заголовок, **крупный заголовок-хук на русском
  (точное написание!)** + оранжевый подзаголовок. Полный шаблон — `pipeline/cover_nanobanana.md`.
- **Реальный лого ОБЯЗАТЕЛЕН:** после генерации композитить настоящий `logo_hd.png` поверх
  (PIL) — крисп, красиво вписанным. Обложка = ИИ-фон+текст + реальный лого.
- Обложка ставится первым кадром ролика (0.30 с).
- **Карусели** (по контент-плану ~1 раз в неделю, 4:5): тоже через `nano_banana_pro`,
  `aspect_ratio:'4:5'`, по слайдам — единый бренд-стиль, вирусный/полезный, с воронкой.
- **Экономия:** Higgsfield — ТОЛЬКО обложка и карусели. Контент роликов (кадры, 3D,
  инфографика, анимации) — бесплатные локальные инструменты, кредиты не жжём.

## ПАЙПЛАЙН

### 1. Сценарий → сегменты озвучки
5–6 предложений (s1..sN). Числа **прописью**, без длинных тире и «не X а Y».
**НИКАКИХ ручных ударений** (ни `+`, ни U+0301, ни RUAccent) — они ЛОМАЮТ
произношение. Чистый текст = верный текст. Подробности → `reference/VOICE_AND_AUDIO.md`.

### 2. Озвучка — edge-tts `ru-RU-DmitryNeural`, ЧИСТЫЙ текст (БЕСПЛАТНО)
Голос по умолчанию `ru-RU-DmitryNeural` (муж., живой, ставит ударения САМ),
`rate="+6%"`. Генерировать **посегментно** (`vo/sN.mp3`) — длительности сегментов нужны
для караоке и точек монтажа. Тайминги слов — пропорционально длине слов внутри сегмента
(см. `plan.py`); можно и `boundary="WordBoundary"`. Отклонённые движки
(Silero/XTTS/Seed Audio/RUAccent) и правила музыки/SFX → `reference/VOICE_AND_AUDIO.md`.
Self-QC: faster-whisper по итогу (ловит слова, не ударения).

### 3. Кадры — УНИКАЛЬНЫЕ, по битам сценария (Закон 2)
- **Pexels API** (`PEXELS_API_KEY`) главный; **Pixabay** (`PIXABAY_API_KEY`) запасной.
  Вертикальные: `orientation=portrait&size=medium`, фильтр `height>width & height>=1900`.
- **1 клип = 1 шот. 10–14 уникальных на ролик.** Запрос на каждый бит сценария свой.
  Качать 5–6 кандидатов на запрос → tile-грид (fps=1) → выбрать глазами лучший.
- **Дедуп по video id** через `reference/USED_FOOTAGE.md` — не брать уже занятые.
- **Бренд-кадры (опционально, под нишу клиента):** если заданы — запросы берём из
  `BRAND_PROFILE.brand_shot_queries` ТЕКУЩЕГО клиента (под его бизнес). Не переносить запросы
  бренд-кадров с другого клиента — ниши разные.
- Хелпер-скелет: `pipeline/fetch_pexels.py` (расширять на уникальность + реестр).
- Звук/музыка — **Freesound API** (`FREESOUND_API_KEY`), `previews.preview-hq-mp3`,
  фильтр `duration:[45 TO 180]`, свой трек и свои SFX на ролик (мимо реестра).

### 4. Анимации — ВЫБИРАТЬ РАЗНЫЕ под смысл, создавать заново (Законы 1,3)
Полный каталог с кодом — `reference/EFFECTS_CATALOG.md`. Движки (в `pipeline/`):
- **`motion/fx_engine.js` (+ fx_page.html)** — палитра брендовых alpha-оверлеев (camera-UI,
  стат-карты, донат, бары, callout, DM/лайки/toast, штампы, кинетика-слэм, партиклы,
  тикер, световой свип). Это ПАЛИТРА для ВЫБОРА и ПАРАМЕТРИЗАЦИИ — не дефолтный набор.
  В каждом ролике брать ДРУГОЙ поднабор, другие цвета/раскладку/движение; мимо `USED_ANIM`.
- **`motion/lottie_render.js`** — Lottie по ключевым словам сценария (свои запросы каждый
  ролик), рендер в alpha PNG-секвенцию.
- **`three/three_fx.html` + `three_render.js`** — three.js 3D-фигура (кристалл/кольцо/
  капля/тор/икоса и т.д.) с прозрачным фоном и env-отражениями, брендовый цвет по смыслу.
  Бесплатно, локально. НЕ Higgsfield.
- **Инфографика КОДОМ (приоритет)** — `pipeline/infographics/` (HTML/SVG/CSS → Playwright →
  прозрачные PNG-кадры → qtrle-mov → overlay). Готовые шаблоны: тахометр-гейдж, физика
  подвески, счётчик цены + трекер шагов, чек-лист. Красивая, анимационная, БОЛЬШЕ её.
  Каждый ролик — ДРУГАЯ цифра/метафора/форма, переписанная заново (см. README там).
- **Remotion-инфографика** (`motion/infographics.tsx`, `counter_gauge.ts`, `kinetic_type.ts`):
  счётчик/одометр/график роста/пончик/бар/слайдер «до-после».
  **Каждый ролик — ДРУГОЙ тип под конкретную цифру, написанный/настроенный заново.**
- **gl-transitions (125)** — `motion/transitions_gl.cjs`, свой набор на ролик, мимо реестра.
- **1–2 беспоук-графики под метафору** сценария (тонущая линия / растущие ярусы / шкала
  «грязь→чисто» и т.п.) — **пишутся с нуля именно под этот ролик**, не переиспользуются.
- Ещё: 2.5D-параллакс фото (Depth-Anything ONNX, локально), rembg-коллаж, световые лики
  (`@remotion/light-leaks`), ч/б-панч на ударном моменте.

Все анимации **привязывать к словам озвучки** (тайминги из json шага 2), **не накладывать
внахлёст** (разносить по времени), плотность — «почти всегда что-то поверх кадра».

### 5. Обложка — Nano Banana Pro (см. блок выше), первым кадром 0.30 с.

### 6. Сборка — ТРИ ЭТАПА ffmpeg (`pipeline/build_reel.py`)
1. **stage1**: обложка + шоты (чередование zoom/pan + chromashift-глитч на склейках) +
   вотермарк + прогресс-бар + грейд + финалка с CTA. Шоты `SEQ:<dir>` = PNG-секвенция.
2. **stage2**: gl-переходы + ВСЕ PNG-оверлеи (`setpts=PTS+T/TB`, `overlay=eof_action=pass`).
   Плотное расписание из манифеста. Блик/лик — `format=gbrp`+`blend=screen`.
3. **stage3**: караоке-ASS + аудио-микс.
Всем `-loop 1` картинкам `-framerate 30` и конечный `-t`.

### 7. Караоке (бренд-константа, НЕ меняется)
Союз Гротеск Bold 76, строчные, 1 строка 2–3 слова (≤16 симв), появление по слову.
Активное слово — **бренд-акцент из лого** (V.CODE: оранж `#EA5920` = ASS `&H2059EA&`),
прошлые белые. НЕ зелёный. Мягкая тень + слой свечения (`\blur14`).

### 8. Звук
Голос: adelay+dynaudnorm. Музыка: 0.13–0.15 + `sidechaincompress` (дакинг). SFX: РАЗНЫЕ
whoosh на переходы + смысловые. Мастер: `loudnorm I=-14:TP=-1.5`.

### 9. QC-чеклист (перед отправкой — глазами по сетке 12–15 кадров)
- [ ] Кадры уникальны (ни один клип не повторяется), меняются каждые 3–5 с.
- [ ] Бренд-кадры (если заданы в профиле клиента) — про нишу именно этого клиента, не с чужого проекта.
- [ ] Анимации разные, НЕ повторяют прошлый ролик (сверка с `USED_ANIM`).
- [ ] Инфографика/беспоук написаны под смысл, не шаблон.
- [ ] Обложка — ИИ (Nano Banana), текст читаемый и без ошибок.
- [ ] Субтитры не наезжают, оверлеи не внахлёст, цвет не фиолетовый, 3D в кадре.
- [ ] Хук в первую секунду, звук сведён (loudnorm), размер < 30 МБ (для отправки).
- [ ] Дописаны реестры `USED_FOOTAGE` и `USED_ANIM`.

### 10. Описание поста
На каждый ролик генерировать текст поста: заголовок-хук + польза + «Сохрани» + CTA
(«напишите СЪЕМКА в директ») + бренд-строка + хэштеги. Отдавать в блоке-копирке.
