# OKO Content Factory — единый контент-завод под ключ

Это НЕ монтажный скрипт. Это оркестратор полного цикла: **анализ → сценарий → сборка →
публикация → аналитика → обучение → усиление**. Работает как живой SMM-специалист,
только 24/7 и без выгорания.

Механика едина для любого клиента. Меняется только один файл — `BRAND_PROFILE.md`
клиента. Всё остальное (законы разнообразия, motion arsenal, пайплайн монтажа,
аналитический цикл) — общее.

## 🎯 ЦЕЛЬ СКИЛЛА В ОДНОЙ ФРАЗЕ

За `N` дней вывести страницу клиента на его цель (например: 100М просмотров на 500
роликов + 20K подписчиков) за счёт **ежедневного** цикла: анализ → сборка → постинг →
аналитика → обучение → усиление залетевших форматов. Никаких повторов, ничего
статичного, всё живое и обучающееся.

## §0. ОНБОРДИНГ КЛИЕНТА (один раз, до первого ролика)

Скилл универсальный — работает под любую нишу. Перед запуском заполнить
`clients/<slug>/BRAND_PROFILE.md`. Если его нет — сначала уточнить у клиента:

1. **Название бренда / бейдж**, ниша, продукт/оффер, УТП.
2. **Логотип** — `logo_hd.png` (PNG с прозрачностью). Из него берём акцентные цвета.
3. **Палитра** — акцентные цвета из логотипа (идут в оверлеи, субтитры, обложку).
   Никогда не «дефолтный зелёный», если он не в бренде.
4. **Шрифт субтитров** (по умолчанию Союз Гротеск/Unbounded/Onest — под бренд).
5. **Голос озвучки** (муж/жен) — edge-tts модель.
6. **CTA** и бренд-строка финалки.
7. **Бренд-кадры** — ключевые слова стока СТРОГО под бизнес клиента (его продукт/склад/
   процесс/доставка), опционально ≥2 на ролик. НИКОГДА не кадры видеосъёмки/оператора/камеры/
   монтажа/студии — это бренд видеопродакшна, а не клиента (частая ошибка утечки из чужого проекта).
8. **Аккаунты** — доступы IG/TT/YT (логин, refresh_token YouTube, Hooppy page_id).
9. **Целевые метрики** — сколько роликов/день, к какой дате, KPI по просмотрам/подписке.
10. **Уведомления** — Telegram bot token + chat_id для алертов и утреннего отчёта.
11. **Higgsfield Ultra** — есть/нет (для обложек через `nano_banana_pro`).

Данные клиента и доступы — в `OKO_MASTER_VAULT.md` (см. раздел 8 у существующих —
Tappio, DIESEL, EKAT, добавлять новых туда же). НИКОГДА не хардкодить пароли в скилл.

Формат `BRAND_PROFILE.md` — см. `templates/BRAND_PROFILE.template.md`.

## §1. ⚖️ ТРИ ЗАКОНА РАЗНООБРАЗИЯ (главное — читать перед каждым роликом)

**Определение:** «наложение» = ВСЁ, что накладывается поверх стокового кадра:
GLB/3D, 3D-графика, инфографика кодом, кинетика, callout, штампы, партиклы,
шейдерные переходы, оверлеи, WebGL, Spine, Lottie, WebP-анимации, GIF, CSS-анимации,
3D/5D-эффекты, свет.

### Закон 1 — всё под сценарий, всё заново

Каждый ролик анализируется отдельно: ниша, тон, метафоры, цифры, боли, оффер.
Из анализа рождаются: кадры, наложения, инфографика, беспоук-графика, музыка,
SFX, грейд, обложка.

- Инфографику и беспоук-графику **пишем ЗАНОВО** под конкретную цифру/метафору.
  Запрещено брать прошлую и менять только цифры/буквы.
- Ассеты (Lottie, gl-переходы, музыка, SFX, кадры) **ищем/скачиваем заново**
  под ключевые слова этого сценария.
- **Неизменны только бренд-константы** (BRAND_PROFILE): голос, шрифт+цвет
  субтитров, лого, финалка, палитра. Остальное — всегда разное.

### Закон 2 — уникальные кадры, темп 3 сек

- **10–14 УНИКАЛЬНЫХ клипов** на ролик (не 3–5 по кругу).
- **Кадр меняется каждые 3 секунды** (иногда 3–5 сек по смыслу).
- **1 бит сценария → 1 запрос → 1 клип.** Последовательность логичная:
  хук → проблема → процесс → результат/цифры → CTA.
- **Между роликами кадры не повторяются** — дедуп по video_id через `USED_FOOTAGE`.
- Опционально **≥2 бренд-кадра** — ТОЛЬКО из бизнеса клиента. НИКОГДА не кадры видеосъёмки/
  камер/монтажа/студии в клиентском ролике (это бренд видеостудии). Нет своих — не вставлять.

### Закон 3 — память запрещает повторы

- Перед сборкой прочитать `USED_FOOTAGE.md` и `USED_ANIM.md`, исключить занятое.
- Один приём/ассет — не чаще 1 раза в 3–5 роликов.
- Переходы и финалы — всегда новые.
- После сборки дописать использованное в оба реестра.

### Красные флаги (значит нарушил закон)

- В двух роликах подряд один набор оверлеев/переходов/3D-фигуры → БРАК.
- В ролике < 8 уникальных клипов ИЛИ клип повторяется → БРАК.
- Инфографика та же, поменяны только цифры → БРАК.
- Обложка из стока + PIL, а не сгенерирована нейросетью → БРАК.
- В клиентском ролике кадры видеосъёмки/камер/монтажа/студии (это НЕ бизнес клиента) → БРАК.

## §2. 🎬 MOTION ARSENAL — законы монтажа

### 2.1 Главные законы (жёстко, нарушение = БРАК)

1. **НУЛЬ СТАТИКИ.** Всё двигается всегда. Все вставки — ВИДЕО-стоки (не фотки).
   3D крутится всю сцену. У любой вставки есть движение: видео само, либо
   ken-burns/пан (crop с `t`), либо вращение 3D, либо анимация графики.
   **Замерший кадр = БРАК.**
2. **ФОРМА КАЖДОЙ ВСТАВКИ УНИКАЛЬНА.** 3 оси без повтора подряд: тип графики
   × раскладка (place) × форма (fmt). Формы во всём ролике стараемся не повторять.
3. **ЛИЦО НЕ ПЕРЕКРЫВАЕМ.** Графика в центре → лицо РАЗМЫТО за ней. Сбоку →
   человек отодвигается (reframe) в противоположную сторону. Никогда сырую
   графику на резкое лицо.
4. **КОНТРАСТ = ЗАТЕМНЕНИЕ ВИДЕО**, а не тёмная плашка. Под графикой затемняем/
   размываем само видео (по зоне или весь кадр). Никаких квадратных полупрозрачных
   подложек — они читаются как глюк. Обводка/свечение у графики — да, плашка — нет.
5. **МИКС форматов:** часть стоков ФУЛЛ-ЭКРАН (с пан/зумом), часть — в формах.
   Не всё в формах, не всё фулл-скрин, не всё «окно справа».

### 2.2 Раскладки (place) — как человек и графика делят кадр

| place | описание |
|---|---|
| `fs` | фулл-экран графика |
| `center` | крупно по центру над размытым/затемнённым фоном |
| `bottom` | крупно снизу (лицо сверху чистое) |
| `left`/`right` | человек отодвинут, графика в затемнённой зоне сбоку |
| `split` | жёсткое деление кадра пополам |
| `lowbar` | нижний титр у самого низа (имя/CTA) |

Затемнение зоны — feather-градиенты (`grad_left/right/bottom.png`, мягкие).
Центр — `gblur=sigma≈15,eq=brightness=-0.34`.
Reframe человека: `scale=2208:1242,crop=1920:1080:X:100`.

### 2.3 Библиотека форм вставок (15+ форм)

Каждая форма — маска ffmpeg (серый feather PNG) + акцентная обводка-кольцо:

**circle, oval, hexagon, pentagon, diamond, roundsquare, rrect, arch, tilt (наклон),
tv, parallelogram, strip (верт. полоса), band (гор. полоса), phone (мокап телефона),
smallcircle** + `full` (фулл-экран) и `feather` (крупно с растворением краёв).

Композит: `media(cover w×h) → alphamerge(mask) → overlay(x,y) на затемнённый фон
→ поверх overlay ring`.

**Фулл-экран движение — ТОЛЬКО crop-pan (4 направления):**
```
scale=2160:1215, crop=1920:1080:x='(iw-1920)*t/DUR':y=...
```
`zoompan` с `zoom-out` или пан-выражением даёт ЧЁРНЫЙ кадр — не использовать.

Генератор форм: `pipeline/motion/make_shapes.py` (PIL: rounded_rect / ellipse /
polygon + GaussianBlur feather; кольцо = outer_mask − inner_mask, залитое акцентом).

### 2.4 Движок графики (`anim.js`) — беспоук-инфографика кодом

Детерминированный движок: `window.renderT(t)` гоняет TRACKS, easing, живые фоны,
reframe, глитч, зерно. ~34 компонента:

`stat, gauge, donut, linechart, map_routes, arrows, flowtree, timeline, compare,
glassticker, glassquote, kinetic, title, name (нижний титр), device (телефон
с экраном), pill, list*, subscribe, cta_title, photo3d, hex/split/card/billboard
(картинка в форме)`.

**Over-mode:** `P.over=true` → графика прозрачная в зоне (opos L/C/R, oscale, oy),
`window.__NOBG` глушит фон. **Подложку-карточку НЕ рисуем** (глюк) — контраст
даёт компоузер затемнением видео.

**Рендер в webm с альфой:** Playwright + headless_shell → скриншоты каждого
кадра с `omitBackground:true` → `ffmpeg -c:v libvpx-vp9 -pix_fmt yuva420p
-auto-alt-ref 0`.

**КРИТИЧНО:** при чтении такого webm в компоузере — `-c:v libvpx-vp9` **ПЕРЕД**
`-i`, иначе нативный vp9-декодер ffmpeg выкидывает альфу (графика станет чёрным
прямоугольником).

Инфографика **пишется заново** под цифру/метафору (SVG `getPointAtLength` для
линий/путей, анимация счётчиков, стрелки, диаграммы). Самая заходящая —
анимированный линчарт «потолок → пробиваем». Таких — побольше.

### 2.5 3D (GLB) — реальные крутящиеся фигуры

**Источники GLB:**
- Higgsfield `generate_3d` (tripo — надёжно, платно; sam_3_3d дешевле, падает
  на тёмных рендерах);
- **MIND MCP** (`start_model_generation` + `animate_generated_model`);
- Sketchfab (`SKETCHFAB_API_KEY`).

**Децимация тяжёлых GLB:**
```
npx @gltf-transform/cli simplify --ratio .25 --error .002
npx @gltf-transform/cli resize --width 768
```
56MB → ~14MB, качество цело.

**Рендер:** three.js r160 ESM + GLTFLoader в `glb_view.html`, отдаётся через
`python3 -m http.server` (ES-модули требуют http, не file://), Playwright
`--use-gl=swiftshader`. Студийный HDR-энвайрон, акцентный+холодный свет,
ACESFilmic. → webm с альфой.

**Правила 3D:**
- Центрировать (offX=0) ИЛИ аккуратно сбоку.
- Влезать в кадр (не обрезать справа).
- Крутиться ВСЮ сцену (без заморозки).
- Уходить плавно (без резкого исчезновения).
- Каждую фигуру — максимум 1 раз за ролик.

### 2.6 WebGL / Spine / Lottie / шейдерные переходы

- **WebGL-эффекты:** React Bits / UIverse / 21st.dev / Three.js shaders —
  скачивать под тему, рендерить в webm с альфой тем же Playwright-пайплайном.
- **Lottie:** lottiefiles (бесплатно), рендер lottie-web в canvas → webm-альфа.
- **Spine/Rive:** rive-canvas / spine-player → webm-альфа. Оверлеи-акценты.
- **Шейдерные переходы:** `pipeline/motion/transitions_gl.cjs` (gl-transitions,
  125 переходов). Для перехода «человек → фулл-экран сток» — быстрый
  zoom-punch/whip/luma.

### 2.7 Звук — музыка по секциям + осмысленные SFX

- **Музыка:** 4–5 треков по секциям (интро/корп/дарк/аплифт/CTA), кроссфейды,
  `volume≈0.28`, даккинг под голос:
  `sidechaincompress(threshold=0.06:ratio=6)`.
- **SFX по смыслу бита** (не просто вжух везде):
  - `impact` на цифрах/сравнениях/3D;
  - `ding` на подписке/CTA/позитиве;
  - `whoosh` на стоках/переходах/появлениях.
  Freesound API (`FREESOUND_API_KEY`).
- **Голос:** `dynaudnorm` + лимитер. Мастер: `amix` + `alimiter=limit=0.96` +
  `loudnorm I=-14:TP=-1.5`.

### 2.8 Грейд / финал

- База уже с грейдом (breathing zoom + grade + vignette + grain) — **НЕ грейдить дважды**.
- Если нужен доп.грейд: `eq + colorbalance + vignette` (лёгкое зерно `noise` — дорого).
- 4K: `scale=3840:2160:flags=lanczos`. Апскейл 1080→4K не даёт деталей, но
  даёт «4K»-лейбл (если требуется).

### 2.9 Чек-лист перед сдачей ролика

- [ ] Нет статики — всё двигается (видео, 3D-спин, пан, анимация).
- [ ] Все вставки — ВИДЕО-стоки (не фотки).
- [ ] Формы уникальны, обводки чёткие, нет квадратных подложек-глюков.
- [ ] Лицо нигде не перекрыто; контраст через затемнение видео.
- [ ] Микс: часть фулл-экран, часть в формах; нет монотонного «окно справа».
- [ ] Нет чёрных кадров (скрин-скан яркости <10 = баг).
- [ ] Губы в синхроне (контиг-тайлинг + посегментный голос).
- [ ] Музыка слышна, SFX по смыслу, финалка/бренд на месте.
- [ ] Длина 15–45 сек, темп кадра 3 сек.
- [ ] Обложка — ИИ (Nano Banana Pro), текст читаемый, без ошибок.
- [ ] Дописаны реестры `USED_FOOTAGE` и `USED_ANIM`.

### 5.3 Озвучка с тайм-кодами слов (edge-tts, бесплатно)

```python
# pipeline/tts_generate.py
import edge_tts, asyncio, json

async def voice(text, voice, out_mp3, out_json):
    comm = edge_tts.Communicate(text, voice, rate="+8%")
    words = []
    with open(out_mp3, "wb") as f:
        async for chunk in comm.stream():
            if chunk["type"] == "audio":
                f.write(chunk["data"])
            elif chunk["type"] == "WordBoundary":  # КРИТИЧНО!
                words.append({
                    "w": chunk["text"],
                    "t": chunk["offset"] / 1e7,
                    "d": chunk["duration"] / 1e7
                })
    with open(out_json, "w") as f:
        json.dump(words, f, ensure_ascii=False, indent=2)
```

**Правила текста:**
- Числа прописью.
- Без длинных тире.
- Без «не X, а Y» (звучит по-нейронному).
- Ударения через U+0301: те́сто, догово́р.
- Ретраи 4–5 раз (edge-tts моргает).

### 5.7 Сборка — три этапа ffmpeg

`pipeline/build_reel.py`:

1. **Stage 1 — база:** обложка + шоты (чередование zoom/pan + chromashift-глитч
   на склейках) + вотермарк + прогресс-бар + грейд + финалка с CTA.
2. **Stage 2 — переходы + оверлеи:** gl-переходы + ВСЕ PNG/webm-оверлеи
   (`setpts=PTS+T/TB`, `overlay=eof_action=pass`). Плотное расписание из
   манифеста. Блик/лик — `format=gbrp` + `blend=screen`.
3. **Stage 3 — субтитры + аудио:** караоке-ASS + аудио-микс (голос + музыка
   с даккингом + SFX + мастер-лимитер `loudnorm I=-14:TP=-1.5`).

**Правила стабильности:**
- Каждый сегмент = отдельный `seg_%03d.mp4` (1920×1080 или 1080×1920, 30fps,
  yuv420p, libx264, единые параметры) → `concat demuxer`. Надёжно, параллелится,
  без рассинхрона.
- **Контиг-тайлинг:** сегменты покрывают таймлайн без дыр (`t0[0]=0`,
  `t1[i]=t0[i+1]`, последний = длине). Иначе склейка «сожмётся» и звук уедет.
- Голос — посегментно (вырезать `cc[t0:t1]` и склеить) → всегда совпадает с видео.
- **Длинные ffmpeg реапаются через 2–3 мин** в фоновых задачах → резать на куски
  ≤90 сек контента или гонять в foreground.
- **Быстрый seek:** перекодировать базу с частыми ключевыми кадрами
  (`-g 15 -keyint_min 15`).

### 5.8 Караоке-субтитры (бренд-константа)

```python
import requests

def schedule_reel(page_id, video_path, cover_path, caption, publish_at_iso, token):

            "caption": caption,
            "publish_at": publish_at_iso,  # 2026-07-17T09:00:00+03:00
```

## §8. 🚀 УСИЛЕНИЕ ЗАЛЕТЕВШИХ ФОРМАТОВ

**Правило:** если ролик набрал ≥ 3× среднего по каналу — это **залёт**, формат
идёт в очередь усиления.

### 8.1 Детекция залёта

```python
# pipeline/detect_hit.py
def is_hit(video_stats, channel_avg, ratio=3.0, min_views=10000):
    return (
        video_stats["views"] >= max(channel_avg * ratio, min_views)
        and video_stats["engagement_rate"] > channel_avg_engagement * 1.5
    )
```

### 8.2 Что делаем с залётом

1. **Полный разбор формата:**
   - Хук (первая строка, первый кадр).
   - Структура (сколько битов, тайминг, разрыв).
   - Раскадровка (типы кадров: close-up, wide, action, text-heavy).
   - Наложения (типы, плотность, чем цепляют).
   - Звук (музыка, SFX, паузы).
   - Обложка (композиция, цвет, лицо/объект).
2. **Записываем шаблон формата** в `analytics/hit_formats/YYYY-MM-DD-<slug>.md`.
3. **Планируем серию продолжений** — 3–5 роликов в том же формате, но на разные
   темы. Пока формат не перестанет залетать (2 подряд ниже среднего = закрываем).
4. **Каждый ролик серии** — свой сценарий, свои кадры, свои наложения (законы
   разнообразия действуют, не копи-паст).

### 8.3 Формат `hit_formats/*.md`

```markdown
# Формат: «hook_stat_bang» — залетел 15.07.2026

## Метрики оригинала
- Просмотров: 2.4M
- Engagement: 12.3%
- Удержание: 78%

## Структура
- 0.0–0.8 сек: хук (крупное число + вопрос)
- 0.8–3.0 сек: паттерн-разрыв (неожиданный факт)
- 3.0–20.0 сек: раскрытие в 3 бита с инфографикой
- 20.0–25.0 сек: моральная закладка + CTA

## Что цепляет
- ...

## Очередь продолжений (в работе)
- [ ] 17.07 — тема Б
```

## §12. 🪲 ГРАБЛИ И ЭКОНОМИЯ

### 12.1 Грабли (проверено болью)

- **edge-tts моргает** — ретраи 4–5 раз; без `WordBoundary` тайминги пустые.
- **Mixkit music = AccessDenied** → музыка с Freesound.
- **Higgsfield-коннектор флапает** — генерацию обложки ретраить, id забирать
  позже через `job_display`/`show_generations`.
- **Nano Banana Pro** иногда пишет модель как `nano_banana_2` — это ок,
  результат тот же.
- **Remotion:** `--gl=swiftshader` (обычная графика), `--gl=angle`
  (light-leaks). Вывод — АБСОЛЮТНЫЙ путь без расширения при `--sequence`.
- **three.js модуль** тянет `three.core.min.js` — класть рядом; ES-модули
  требуют `python3 -m http.server`, не `file://`.
- **Playwright chromium копит память** — рендерить оверлеи батчами (≤12) со
  свежим браузером, иначе OOM на ~15-й задаче.
- **Отдача 30MB-лимит в чат** — хостить видео: Higgsfield `media_upload` → PUT
  байты curl'ом (`--cacert /root/.ccr/ca-bundle.crt`) → `media_confirm` →
  CDN-ссылка `d2ol7oe51mr4n9.cloudfront.net`.
- **Фоновые ffmpeg реапаются** через ~2–3 мин — резать на куски ≤90 сек
  контента или гонять в foreground кусками. 4K — по 50 сек, concat demuxer.
- **Быстрый seek базы:** перекодировать базу с частыми ключевыми кадрами
  (`-g 15 -keyint_min 15`) → seek мгновенный.
- **Gemini как «глаза»:** `gemwatch.sh` через curl (не urllib — иначе CF 1010);
  ротация ключей на 429; при 503 модель перегружена, повторить.
- **webm с альфой:** при чтении в компоузере `-c:v libvpx-vp9` ПЕРЕД `-i`,
  иначе альфа выкидывается → графика чёрным прямоугольником.
- **zoompan с zoom-out или пан-выражением даёт чёрный кадр** — использовать
  только crop-pan.
- **IG Reels через Graph API** требует бизнес-аккаунт + связку с FB Page,
  иначе только Hooppy или Playwright.

### 12.2 Экономия (жёстко)

- **Higgsfield-кредиты — ТОЛЬКО обложка и карусели** (Nano Banana Pro).
  Ролики (кадры, 3D, инфографика, анимации) — бесплатные локальные инструменты.
- **Стоки — Pexels/Pixabay** (бесплатно), 4K приоритет.
- **Озвучка — edge-tts** (бесплатно, MS Azure).
- **Музыка/SFX — Freesound** (бесплатно с атрибуцией через API).
- **3D — Sketchfab бесплатные модели + генерация MIND MCP** (freemium).
- **Транскрипт — Gemini 2.5 Flash** (бесплатный лимит) или локальный Whisper.
- **Higgsfield generate_video — только когда стоков реально нет.**
  Проверять `balance` перед вызовом.
- **fal.ai** — резерв, баланс 0 в дефолте.

### 12.3 Правила безопасности данных

- **Секреты хранить только в `secrets.env` / VAULT.** Никогда не коммитить в git.
- **Токены YT/IG клиентов** — только в `OKO_MASTER_VAULT.md` (secrets.env облака).
- **Не давать клиентские пароли в чат** — только `client_slug` +
  «доступы взяты из VAULT».
- **Не хранить видео клиентов дольше 30 дней** на бесплатных CDN — переносить
  в клиентский R2/S3 (у OKO есть Cloudflare R2 + Twcstorage S3).

## §14. 📌 ЕДИНАЯ ФИЛОСОФИЯ (то, что нельзя забыть)

- **Один процесс, а не разовая нарезка.** Анализ → сценарий → сборка →
  публикация → аналитика → обучение → усиление. Каждый день. Замкнутая петля.
- **Ничего статичного, никогда.** Всё движется всегда. Замерло = брак.
- **Каждый ролик — с нуля под сценарий.** Наложения, кадры, музыка,
  обложка — новые. Реестры-память запрещают повторы.
- **Кадр меняется каждые 3 секунды.** Наложение — каждый кадр или через кадр.
- **Ролики 15–45 секунд.** Дольше — падение просмотров.
- **Всегда 4K стоки** (если есть на Pexels/Pixabay).
- **Анализ конкурентов перед КАЖДЫМ роликом.** Минимум 5 роликов от 1M
  просмотров, транскрипт, раскадровка, комментарии — всё.
- **Прогон сценария через все скиллы** (humanizer, marketing, hook, media).
- **Утренний отчёт в бота — 10:00 МСК каждый день.**
